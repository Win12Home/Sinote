## Pull request merge requirements
Your **pull request** will be reviewed and checked by us, you can **follow** these rules.
1. Passed CI/CD and haven't got any malicious code or un-security things
2. It is approved by me or 2 maintainers. **(WE'RE LOOKING FOR MAINTAINERS BRO)**
3. This change is not already included in the latest version or the `dev` branch.
4. This change is **beneficial** to our project. Such as **IMPROVE**, **OPTIMIZE WITH C** and similiar.
## Issue report requirements
If you **found** a bug or **have got a good idea**, please report to the issue page that the premise is following these below this. ‚Üì‚Üì
1. There isn't the same **issue** as yours if you **aren't** report a **CRITICAL** or **FATAL ISSUE**.
2. This **issue** hasn't already been fixing in the **latest version**.
## Fix issue that happened
If you **want to** fix some **issue** in **GitHub Issue Page**, please follow these rules.
1. Fix the **CRITICAL or FATAL** issue first. We will **thank you** when you finished this issue fixing and pull request to **Sinote**!
2. Fix or answer the **Frequently Asked Issue or Question** second.<br>

> [!NOTE]
> These are guidelines, not strict rules. You are not required to follow them exactly.
